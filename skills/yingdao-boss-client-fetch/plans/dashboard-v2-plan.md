# 看板 V2 方案

> 基于当前 `_template.html`（GPT 实现的双模式版本）的增量优化方案。
> 目标：从"客户状态看板"升级为"CSM 今日工作台"，同时保持指标观察模式的客观数据纯粹性。
> 约束：保持单 HTML 文件 inline 数据的形态，方便无技术背景的 CSM 通过飞书直接接收使用。

## 一、本次改动清单（按模块组织）

### 改动 1：对比描线叠加（指标观察模式）

**问题**：当前选择"环比/同比"后，卡片只显示一个孤立的 `+65.5%`，图表里只有当期一条线，CS 无法直观看到对比基线长什么样。

**方案**：

- 当期数据：保留现状（主色实线 + 渐变填充）
- 对比期数据：同色系浅色虚线，仅描线不填充
  - 颜色：当期色 + 透明度 0.4
  - 线型：`dashStyle: "dashed"`
  - 不画 `areaStyle`
- X 轴对齐方式（关键，决定视觉语义正确性）：
  - **环比**：对比期数据在 X 轴上**左侧延伸**，与当期不重叠。即当期 30 天在右侧，上一同长度 30 天在左侧，中间无间隙。X 轴标签用各自实际日期（不混用同一组标签）
  - **同比**：对比期数据**重叠在同一 X 位置**，两条线在同一日期坐标上对比。X 轴用当期日期标签
- tooltip：同时显示当期值、对比期值、变化率
  - 环比模式：tooltip 跟随鼠标，分别显示当前日期和对应的历史日期
  - 同比模式：tooltip 同一日期下显示"今年"和"去年"两条数值
- 无对比期数据时（如新签客户没有去年同期数据）：
  - 图表区域右上角标注"无同期数据"水印
  - 卡片对比 stat 显示"无同期数据"（保留现状）

**影响文件**：`_template.html` 的 `renderClientChart` 函数

### 改动 2：指标定义折叠区

**问题**：9 个指标的聚合方式（sum/avg/last/derived）和对比口径不同，CS 容易误读"+50%"在不同指标上的含义。

**方案**：

- 位置：看板页脚上方，footer 之前
- 默认状态：**折叠**，只显示标题"指标定义与对比口径说明"和一个展开箭头
- 展开后内容：一张表格，列出所有指标的聚合方式 + 环比/同比公式

**表格内容**：

| 指标 | 聚合方式 | 环比公式 | 同比公式 |
|------|---------|---------|---------|
| 运行次数 | 区间内每日值求和 | (当期和 − 上一同长度周期和) / \|上一周期和\| | (当期和 − 去年同期和) / \|去年同期和\| |
| 运行时长 | 区间内每日值求和（秒） | 同上 | 同上 |
| 活跃应用数 | 区间内每日值取平均 | (当期均值 − 上期均值) / \|上期均值\| | (当期均值 − 去年同期均值) / \|去年同期均值\| |
| 新建应用数 | 区间内每日值求和 | (当期和 − 上期和) / \|上期和\| | 同理 |
| 高级账号活跃 | 区间内每日值取平均 | 同活跃应用数 | 同理 |
| 账号饱和度 | 区间内每日值取平均 | 同上 | 同上 |
| 健康度 | 区间内每日值取平均 | 同上 | 同上 |
| 高级账号使用率 | 区间最后一天的 已开通/已购额度 | (当期末态比值 − 上期末态比值) / \|上期末态比值\| | (当期末态比值 − 去年同期末态比值) / \|去年同期末态比值\| |
| 有效运行占比 | 区间内 useDayRunDura 之和 / dayRunDura 之和 | (当期比值 − 上期比值) / \|上期比值\| | 同理 |

**不在本次范围**：图表下方不放任何公式说明（避免每个图表重复）

### 改动 4：Top 10 优先级算法修正

**问题**：当前 `expired` 队列优先级 +130，比 `rescue` 还高，导致已过期客户抢占 Top 1-3 位置，而真正还能抢救的临期客户被挤到后面。

**方案**：

1. **expired 队列不进 Top 10**：`topPriority` 函数过滤条件从 `c.diagnosis.queue !== "stable"` 改为 `!["stable", "expired"].includes(c.diagnosis.queue)`
2. **data-check 队列也不进 Top 10**：数据异常属于核查事项，不是 CS 日常跟进动作
3. **rescue 优先级提到最高**，特别是 veryNear（30 天内）

**调整后的优先级基线**：

| 队列 | 基线优先级 | 说明 |
|------|----------|------|
| rescue (veryNear ≤30天) | +130 | 最高，临期且使用危 |
| rescue (near 31-60天) | +110 | 次高 |
| big-swing | +95 | 高价值客户波动 |
| renewal-healthy (veryNear) | +88 | 临期但健康，推进续费 |
| new-onboarding | +82 | 新签未激活 |
| renewal-healthy (near) | +70 | 续费推进 |
| drop | +65 | 使用骤降 |
| health | +58 | 健康度低 |
| followup | +54 | 超期未跟进 |
| personal | +42 | 个人版转化 |
| opportunity | +44 | 机会客户 |
| stable | +12 | 稳定 |
| expired | — | 不进 Top 10 |
| data-check | — | 不进 Top 10 |

**附加权重**（在基线上叠加，保留现有逻辑）：
- sharpDrop: +35
- nearZeroUse: +30
- nearExpiry: +max(0, 60 − daysRemaining)
- followupOverdue: +min(35, floor(daysSinceFollowUp / 10))
- personalSignal 且不在 personal 队列: +8

**影响文件**：`_template.html` 的 `diagnoseClient` 和 `topPriority` 函数

### 改动 7：Y 轴统一量纲（指标观察模式默认开启）

**问题**：不同客户的 Y 轴范围各自自动适应，一个客户 0-10、另一个 0-10000，CS 会下意识比较"线的高度"导致误判。

**方案**：

- 指标观察模式：**默认开启**统一 Y 轴
- CSM 诊断模式：默认关闭（诊断更关注单个客户的趋势变化，量级对比意义不大）
- 控制栏增加一个开关"统一 Y 轴量纲"

**实现逻辑**：
- 遍历当前筛选下的所有客户，取该指标在当前期的最大值 `globalMax`
- 所有图表的 Y 轴 `max` 设为 `globalMax * 1.1`（留 10% 顶部空间）
- 若 `globalMax` 为 0，则不设 max（让 ECharts 自动）

**注意**：对比描线叠加时（改动 1），对比期数据可能超过当期 max，需要在计算 globalMax 时把对比期也纳入（取两者最大值）。

**影响文件**：`_template.html` 的 `renderClientChart` 函数 + 控制栏新增开关

## 三、实现注意事项

### 3.1 对比描线的 X 轴对齐（改动 1 的技术细节）

环比模式的 X 轴拼接：
```
xAxisData = [...previousDates.map(fmtDate), ...currentDates.map(fmtDate)]
seriesCurrent = [null * previousDates.length, ...currentValues]
seriesPrevious = [...previousValues, null * currentDates.length]
```
两条 series 共享同一个 xAxis，用 null 填充对方区域，实现"左历史 + 右当期"的视觉。

同比模式的 X 轴对齐：
```
xAxisData = currentDates.map(fmtDate)  // 只用当期日期
seriesCurrent = currentValues
seriesPrevious = currentDates.map(d => {
  // 找去年同期对应日期的数据
  const yoyDate = addMonths(d, -12)
  return getValueForDate(yoyDate)
})
```
两条 series 在同一 X 位置叠加。

### 3.4 筛选状态管理（改动 3）

```js
const state = {
  // ... 现有字段
  filters: {
    serviceStage: ["本年度新签", "首年续费", "多年续费", "买断使用", "售前阶段", "合作中（异常订单）"],  // 默认不含"已流失"
    customerTier: ["SKA", "KA", "SMB", "未分层"],
    renewalQuarter: ["Q1", "Q2", "Q3", "Q4", "不在续费期"],
    csOwner: []  // 空数组表示全选，运行时从数据动态填充
  }
};
```

筛选逻辑：
```js
function matchesFilters(client) {
  const p = client.profile;
  if (!state.filters.serviceStage.includes(p.serviceStage || "未填写")) return false;
  if (!state.filters.customerTier.includes(p.customerTier || "未分层")) return false;
  if (!state.filters.renewalQuarter.includes(p.renewalQuarter || "不在续费期")) return false;
  if (state.filters.csOwner.length && !state.filters.csOwner.includes(p.csOwner || "未填写")) return false;
  return true;
}
```
