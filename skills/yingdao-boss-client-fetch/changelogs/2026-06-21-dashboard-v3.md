# Changelog: Customer-Success Dashboard V3 (2026-06-21)

## Background

The V2 action dashboard spent too much vertical space on filters before any
client card appeared, forced per-metric tab clicking inside the client drawer,
repeated the same comparison figure in several places, and surfaced system
judgement labels (e.g. `大客户波动`) without explaining why a client matched.
V3 improves first-screen scan efficiency and turns the client drawer into a
multi-metric analysis workspace, while keeping the single self-contained HTML
form so it can still be shared directly through Feishu.

All work is in `dashboard/_template.html`; `dashboard.html` is regenerated by
`build_data.py`.

## Changes Implemented

### 1. First-screen filter collapse
- The four structured filter groups (服务阶段 / 客户分层 / 续费季度 / CS 负责人)
  now live in a collapsible panel that is **hidden by default**, so client cards
  appear higher on the page.
- A `筛选` toggle button was added to the control row. Its label shows a live
  summary — `筛选 · N 组生效` — where `N` counts filter groups that currently
  narrow the result set (a fully-selected group does not count). The default
  rule that hides `已流失` / `已过期` is unchanged, so a fresh load reads
  `筛选 · 1 组生效`.

### 2. Drawer reworked into a multi-metric workspace
- **All 9 metrics at once**: the single metric-tab + single chart was replaced
  by 9 per-metric charts. Each chart header shows the metric name plus its
  current-period aggregate and (when comparing) a coloured change delta.
- **Manual column layout**: a `1 / 2 / 3 列` single-select (default `1`),
  persisted to `localStorage` key `dashboard_drawer_chart_columns`.
- **Wider, draggable drawer**: default width raised from ~560px to ~840px via a
  `--drawer-width` CSS variable; a left-edge drag handle resizes it between
  560px and 90vw, persisted to `localStorage` key `dashboard_drawer_width`.
  Mobile (`max-width: 760px`) stays full-screen with the handle disabled.
- **Independent comparison**: a drawer-only `环比 / 去年同期 / 不对比` selector
  (`state.drawerComparison`) that initialises from the comparison in effect on
  entry (the visible `state.comparison` in 指标观察 mode, `环比` in CSM mode) and
  never writes back to the outer board.
- **Independent Y-axis**: drawer charts always auto-fit each metric's own data
  and never inherit the outer 统一 Y 轴量纲 setting.
- Drawer chart instances are tracked separately and disposed on close to avoid
  ECharts instance leaks across repeated opens.

### 3. Metric-observation de-duplication
- Section title changed to `{指标名} · 从低到高 · {n} 客户` (removing the separate
  `指标观察 · n` + redundant sort description).
- The per-card `指标观察` status pill and the duplicated `环比 +xx%` reason line
  were removed; the comparison figure remains only in the stat area.

### 4. Explainable CSM diagnosis labels
- `diagnoseClient` now returns `explainTitle` (queue definition) and
  `explainDetail` (the client's actual hit basis, built from real values, e.g.
  `命中依据：到期剩余 12 天；运行次数环比 -58.4%；活跃应用环比 +91.7%`).
- These are surfaced as a native `title` tooltip on the diagnosis status pill,
  so a CSM can see both the rule and why this client matched it.

### 5. Top 10 visual unification
- `今日优先跟进 Top 10` now reuses the standard client-card template (via new
  `clientCard(..., { rank, showPriority })` options) instead of a bespoke
  `priority-card`. Cards gain a `#1` rank badge and a `优先级 {score}` tag and
  render the same trend chart / tags / reason / action as queue cards. The old
  `priorityCard` function and `priority-*` styles were removed.

## Out of scope (unchanged)
Bottom detail table, custom tooltip component, drawer follow-up timeline,
step-by-step action guidance, and any multi-file build splitting.

## Verification & Usage

Rebuild from the latest local JSON (no network fetch needed):

```bash
python3 skills/yingdao-boss-client-fetch/dashboard/build_data.py
```

Verified on 2026-06-21 against existing runtime JSON (42 report clients):
- `node --check` on the extracted inline script — syntax clean (template and
  built file).
- Headless Chrome render of the built `dashboard.html` — `init()` runs with
  **no console errors**; 39 metric cards each initialise an ECharts canvas;
  filter panel is collapsed; title reads `… · 从低到高 · 39 客户`.
- Driven CSM-mode + drawer pass — Top 10 client cards with rank/priority,
  diagnosis pill tooltip carrying definition + hit basis, drawer opens at 840px
  with 9 metric charts, 1-column / 环比 defaults, and a working resize handle.
