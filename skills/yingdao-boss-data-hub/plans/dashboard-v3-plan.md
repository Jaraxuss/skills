# 看板 V3 优化方案

> 基于当前 V2 看板继续优化。
> 目标：提升首屏空间利用率、提升客户详情页分析效率、减少重复信息、增强 CSM 诊断解释性。
> 约束：继续保持单 HTML 文件 inline 数据形态，方便直接通过飞书分发。

---

## 一、本次改动范围

### 改动 1：首屏空间利用率优化

**问题**：进入页面后，上半部分主要被筛选项占用，客户卡片露出较少，影响看板作为工作台的扫描效率。

**方案**：

- 保留第一行核心控制：
  - 模式切换
  - 时间段
  - 自定义日期
  - 指标 / 对比（仅指标观察模式显示）
  - 搜索客户名称
  - 统一 Y 轴量纲
  - 卡片字段
  - 筛选按钮
- 四组结构化筛选收进折叠面板：
  - 服务阶段
  - 客户分层
  - 续费季度
  - CS 负责人
- 默认状态：
  - 筛选面板收起
  - 筛选按钮显示当前筛选摘要，例如 `筛选 · 4组启用`
  - 已流失 / 已过期默认不展示的规则保持不变
- 展开后仍保留当前 chip 多选交互。

**设计原则**：

- 首屏优先展示客户卡片。
- 筛选是辅助动作，不应长期占用垂直空间。
- 筛选状态需要可见，但不需要完整铺开。

**影响文件**：

- `dashboard/_template.html`

### 改动 2：详情页改为多指标分析工作区

#### 2.1 默认展示所有指标折线图

**问题**：当前详情页通过指标 tab 单图切换，需要逐个点击，效率低。

**方案**：

- 详情页进入后直接展示 9 个指标折线图：
  - 运行次数
  - 运行时长
  - 活跃应用数
  - 新建应用数
  - 高级账号活跃
  - 账号饱和度
  - 健康度
  - 高级账号使用率
  - 有效运行占比
- 去掉“单指标 tab + 单图”的默认呈现方式。
- 每个指标一张独立图，标题显示指标名称和当前区间聚合值。

#### 2.2 详情图表布局由 CSM 手动选择

**用户确认口径**：

- 默认展示 `1 列`。
- 不根据详情页宽度自动切换列数。
- 详情页内提供单选项，让 CSM 自由选择布局。

**布局选项**：

- `1 列`：默认，单图最宽，适合深度看趋势。
- `2 列`：适合快速横向比较多个指标。
- `3 列`：适合大屏概览。

**持久化**：

- 布局选择写入 `localStorage`。
- key 建议：`dashboard_drawer_chart_columns`

#### 2.3 详情页宽度优化与可拖拽

**问题**：当前详情页宽度约 560px，展示多指标图时不够。

**方案**：

- 默认详情页宽度调整为约当前 1.5 倍：
  - 当前约 `560px`
  - V3 默认约 `840px`
- 详情页左边增加拖拽把手。
- 拖拽范围：
  - 最小：`560px`
  - 最大：`90vw`
- 拖拽后的宽度写入 `localStorage`。
- key 建议：`dashboard_drawer_width`
- 下次打开详情页沿用上次宽度。

**交互细节**：

- 鼠标移动到左边缘显示可拖拽状态。
- 拖拽过程中实时更新宽度。
- 松开鼠标后保存宽度。
- 移动端仍全屏，不启用拖拽宽度。

#### 2.4 详情页独立对比口径

**问题**：外部可以切换环比 / 同比 / 不展示，但进入详情后不能独立修改，需要关闭详情页回外部调整。

**方案**：

- 详情页顶部增加独立对比口径单选：
  - 环比
  - 去年同期
  - 不对比
- 默认值继承进入详情时外部选择。
- 在详情页内修改只影响详情页图表。
- 不反向修改外部看板的对比选择。
- 详情页关闭后，外部状态保持不变。

**状态字段建议**：

```js
state.drawerComparison = state.comparison;
```

打开详情页时初始化；详情页内修改只更新 `drawerComparison`。

#### 2.5 详情页图表独立 Y 轴

**问题**：外部统一 Y 轴量纲会让低量级客户看不出波动。详情页的目标是分析单个客户，应动态调整量纲。

**方案**：

- 详情页图表永远使用独立 Y 轴。
- 不继承外部 `统一 Y 轴量纲`。
- 每张指标图按该指标自己的当前期 + 对比期数据自动适配 Y 轴。

**影响文件**：

- `dashboard/_template.html`

### 改动 3：减少重复信息

#### 3.1 指标观察模式去重复

**当前重复点**：

- 标题左侧显示 `指标观察 · n`
- 标题右侧又显示 `{指标名} 从低到高排序`
- 每张客户卡片右上角显示 `指标观察`
- 卡片文本区再次显示 `环比 +xx%`，而 stat 区已经显示同一信息

**方案**：

- 指标观察主标题改为：
  - `{指标名} · 从低到高 · {n} 客户`
- 去掉指标观察卡片右上角 `指标观察` 标签。
- 指标观察卡片默认不展示 `reason` 文案。
- 指标观察模式卡片保留：
  - 客户名
  - 客户分层 / CS / 服务阶段 / 到期天数 / 最近跟进
  - 当前指标
  - 对比变化
  - 活跃应用
  - 趋势图

#### 3.2 其他建议去重项

需要确认是否一起做：

1. **明细表与卡片重复**
   - 明细表当前展示大量卡片已有字段。
   - 建议保留明细表，因为它支持表格扫描和复制，不作为本次删除项。

2. **CSM 诊断卡片右上角队列标签**
   - 在行动队列分组内，卡片右上角再显示同一个队列名有重复。
   - 但该标签后续要承载 hover 解释，因此建议保留。
   - Top 10 区域没有分组上下文，也需要保留队列标签。

3. **Top 10 与下方行动队列重复客户**
   - Top 10 是今日入口，下方是完整队列。
   - 重复是有意设计，不建议删除。

4. **详情页基础信息与卡片信息重复**
   - 详情页需要独立上下文，不建议删除。

**影响文件**：

- `dashboard/_template.html`

### 改动 4：CSM 诊断标签解释性增强

**问题**：`大客户波动`、`临期健康续费`、`新签未激活` 等标签属于系统判断，不是客观数据。CSM 看到标签后还需要猜判断依据。

**方案**：

- 对所有系统判断标签增加 hover tooltip。
- tooltip 内容包含两层：
  - 队列定义
  - 当前客户命中依据

**示例**：

`大客户波动`

```text
队列定义：SKA/KA 客户核心指标明显波动
命中依据：客户分层=KA；运行次数环比 -32.4%；活跃应用数环比 -18.1%
```

`临期健康续费`

```text
队列定义：60 天内到期且仍保持稳定使用
命中依据：到期剩余 11 天；运行次数 7,595 次；活跃应用 79.0 个
```

`新签未激活`

```text
队列定义：本年度新签客户当前使用不足
命中依据：服务阶段=本年度新签；运行次数 0 次；活跃应用 0.0 个
```

**实现建议**：

- 在 `diagnoseClient` 返回值中新增：

```js
explainTitle
explainDetail
```

- `status-pill` 或诊断 tag 上增加 tooltip。
- 先用浏览器原生 `title` 属性实现，成本低、稳定。
- 如果后续需要更强视觉，再改为自定义 tooltip。

**影响文件**：

- `dashboard/_template.html`

---

### 改动 5：今日优先跟进 Top 10 样式统一

**问题**：Top 10 当前是独立方框样式，与下方客户卡片视觉体系不一致。

**方案**：

- Top 10 改为复用客户卡片样式。
- 保留优先级排序信息：
  - 在卡片左上或标签区显示 `#1`、`#2`
  - 可显示 `优先级 {score}`
- Top 10 卡片仍展示：
  - 客户名称
  - 诊断标签
  - 客户分层 / CS / 服务阶段 / 到期天数 / 最近跟进
  - 诊断原因
  - 建议动作
  - 运行次数趋势图
- 视觉上与下方行动队列客户卡片一致，但多一个 rank 标识。

**实现建议**：

- 扩展 `clientCard(client, metric, chartPrefix, mode, options)`：

```js
clientCard(c, metricDef("dayRunCnt"), "priority", "diagnosis", {
  rank: idx + 1,
  showPriority: true
})
```

- 避免维护两套卡片模板。

**影响文件**：

- `dashboard/_template.html`

## 二、明确不在本次范围

| 项 | 原因 |
|----|------|
| 删除底部明细表 | 表格仍有扫描、排序对照、复制价值 |
| 自定义 tooltip 组件 | 原生 title 可先满足解释性，降低实现复杂度 |
| 详情页跟进记录时间线 | 当前数据源不覆盖 |
| 详情页行动建议步骤化 | 业务规则还需继续沉淀 |
| 看板性能拆包 | 仍保持单 HTML 文件分发 |

---

## 三、建议实施顺序

| 顺序 | 改动 | 理由 |
|------|------|------|
| 1 | 首屏筛选区折叠 | 直接改善进入页面后的空间利用率 |
| 2 | 指标观察去重复 | 小改动，立即降低噪音 |
| 3 | 详情页宽度 + 拖拽 + localStorage | 为多指标图提供空间基础 |
| 4 | 详情页 9 指标图 + 列数单选 | 核心效率提升 |
| 5 | 详情页独立对比口径 + 独立 Y 轴 | 保证详情分析体验完整 |
| 6 | CSM 诊断标签 tooltip | 提升系统判断可解释性 |
| 7 | Top 10 样式统一 | 视觉一致性和维护性优化 |

## 五、实现备注

### localStorage keys

```js
dashboard_drawer_width
dashboard_drawer_chart_columns
```

当前已有：

```js
dashboard_card_fields
```

需要保持兼容，不要覆盖用户已有卡片字段配置。

### 详情页图表渲染

详情页 9 图会明显增加 ECharts 实例数量。由于只打开一个客户详情页，数量仍可控。

关闭详情页时需要 dispose 详情页图表实例，避免重复打开不同客户后实例泄漏。

### 对比口径隔离

外部状态：

```js
state.comparison
```

详情状态：

```js
state.drawerComparison
```

详情图表渲染只读 `drawerComparison`。
