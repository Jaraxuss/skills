# Customer-Facing Copy Rules

Use this reference to turn internal notes into customer-ready slide text.

## Principles

- Write for the customer in the room, not for the presenter preparing the deck.
- State business value before implementation detail.
- Use concrete workflow nouns: report, field, rule, exception, owner, output, archive.
- Prefer short claims that can fit on a slide.
- Put presenter reminders in speaker notes only when needed.

## Replace Internal Phrases

| Internal phrasing | Customer-facing phrasing |
| --- | --- |
| 不要讲成纯编程课 | 围绕业务案例理解进阶数据处理方法 |
| 这里只讲基础认知 | 先建立能力边界和适用场景 |
| 业务人员只需要知道 | 业务侧可以重点关注输入、规则和输出 |
| 这页给讲师提示 | [move to speaker notes] |
| 先把学员预期摆正 | 先对齐自动化链路中的分工边界 |
| AI味道很重的说明性措辞 | [rewrite as concrete business outcome] |

## Good Customer Copy Examples

- 从业务问题出发，看到数据处理结果。
- 把原始报表整理成标准数据，便于后续自动处理。
- 帮助业务与 IT 对齐字段、规则和输出结果。
- 影刀负责流程编排，Python 负责复杂数据处理。
- 处理结果可以继续接回影刀，完成通知、归档和分发。

## Copy QA

Before delivery, scan visible slide text for:

- internal coaching language
- vague promises
- long paragraphs
- duplicate slide titles
- unreadable code comments
- terms that should be explained for business users
