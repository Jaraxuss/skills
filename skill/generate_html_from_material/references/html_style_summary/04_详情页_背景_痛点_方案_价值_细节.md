# 详情页（背景+痛点+方案+价值+细节）风格总结

## 概述

用于作为包含 **需求背景**、**业务痛点**、**解决方案**、**实现流程或细节**、**方案价值** 的案例详情页面。这类页面采用左右分栏布局，信息密度高，适合展示完整的业务场景解决方案。

## 分析的页面

- `pages/custom/超复杂表单识别并填写.html`

## 1. 设计规范

### 1.1 画布尺寸

- **外壳容器**: 全屏 (`w-screen h-screen`)，深蓝背景 (`bg-[#0f172a]`)
- **幻灯片容器**: 固定 1280×720 像素 (`w-[1280px] h-[720px]`)
- **内边距**: `px-14 py-10` (左右 56px，上下 40px)

### 1.2 背景色/渐变

## 2. 布局模板

### 2.1 页面整体结构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  Header (场景标题 + 主标题)                                                   │
├─────────────────────────────┬───────────────────────────────────────────────┤ 
│                             │                                               │
│  左栏 (50%)                  │  右栏 (50%)                                   │
│  ├─ 背景+痛点卡片             │  │
│  ├─ 解决方案卡片              │  │                                            │
│  │  └─ 工作流程图             │  └─ 实现流程或细节展示卡片 (如Excel预览、AI工作流等)│
│  └─ 价值展示区域              │                                               │
│                             │                                               │
└─────────────────────────────┴───────────────────────────────────────────────┘
```

### 2.2 内容区域划分

- **Header 区域**: 相对定位 + z-index:20，收缩不增长 (`shrink-0`)，底部边距 `mb-6`
- **主内容区域**: Flex 布局，水平排列，间距 `gap-8`，自动填充剩余空间 (`flex-1`)
- **左栏/右栏**: 各占 50% 宽度 (`w-1/2`)，Flex 纵向排列，间距 `gap-4`

### 2.3 响应式处理

本页面采用固定尺寸设计，通过 `slide-scale-wrapper` 进行等比缩放适配不同屏幕。

### 2.4 右栏防溢出规范（重要）

在 1280×720 固定尺寸布局中，右栏通常放置图片预览卡片。**Flexbox 子元素默认 `min-height: auto`**，高纵横比图片（如技术规格书、长表单截图）会撑爆容器，导致下方卡片被挤出视口。

**必须遵循以下规则：**

1. **逐层添加 `min-h-0`**：右栏容器、每个卡片、图片容器都必须添加 `min-h-0`
2. **用比例 flex 分配空间**：多卡片使用 `flex-[3]` / `flex-[2]` 按比例分配，禁止混用 `flex-1` + `h-40`
3. **图片容器 `overflow-hidden`**：确保图片被裁切在容器范围内
4. **纵向长图用 `object-cover object-top`**：展示顶部缩略，用户点击 Lightbox 查看全图

```
层级关系示意：
右栏（flex flex-col gap-4 min-h-0）
├── 卡片1（flex-[3] flex flex-col min-h-0）
│   └── 图片容器（flex-1 min-h-0 overflow-hidden）→ img: object-cover object-top
└── 卡片2（flex-[2] flex flex-col min-h-0）
    └── 图片容器（flex-1 min-h-0 overflow-hidden）→ img: object-contain
```

## 3. 组件清单

### 3.1 毛玻璃卡片 (Glass Card)

用于承载主体内容的半透明卡片。

```css
.glass-card {
    background: rgba(255, 255, 255, 0.8);
    backdrop-filter: blur(12px);
    border: 1px solid rgba(255, 255, 255, 0.5);
    box-shadow: 0 8px 32px rgba(0, 0, 0, 0.04);
    border-radius: 16px;
}
```

### 3.2 带侧边条的卡片

卡片左侧带有颜色指示条，悬停时宽度增加。

- **深色条 (slate-800)**: 用于背景/问题描述
- **红色条 (rpa-red)**: 用于解决方案

```html
<div class="glass-card p-4 relative overflow-hidden group">
    <div class="absolute top-0 left-0 w-1.5 h-full bg-slate-800 transition-all group-hover:w-2"></div>
    <!-- 内容 -->
</div>
```

### 3.3 小节标题组件

带图标的小节标题，图标使用圆角矩形背景。

```html
<div class="flex items-center gap-2 mb-2">
    <div class="w-7 h-7 rounded-lg bg-{color}-100 flex items-center justify-center text-{color}-600 shadow-sm">
        <i class="fa-solid fa-{icon} text-sm"></i>
    </div>
    <h3 class="text-sm font-bold text-slate-900 tracking-tight">标题文字：</h3>
</div>
```

**图标颜色规范**：
- 需求背景: `bg-slate-100` + `text-slate-800` + `fa-file-lines`
- 业务痛点: `bg-orange-100` + `text-orange-600` + `fa-triangle-exclamation`
- 解决方案: `bg-red-50` + `text-rpa-red` + `fa-robot`
- Excel文件: `bg-emerald-100` + `text-emerald-600` + `fa-file-excel`
- PDF文件: `bg-blue-100` + `text-blue-600` + `fa-file-pdf`

## 5. 动画约定

### 5.1 入场动画

所有 `.fragment` 元素默认隐藏，通过添加 `.visible` 类显示。

```css
.fragment {
    opacity: 0;
    transform: translateY(30px);
    transition: all 0.8s cubic-bezier(0.16, 1, 0.3, 1);
    pointer-events: none;
}

.fragment.visible {
    opacity: 1;
    transform: translateY(0);
    pointer-events: auto;
}
```

### 5.2 动画步骤控制

使用 `animation_step` 属性控制元素显示顺序：

```html
<div class="fragment" animation_step="1">第一步显示</div>
<div class="fragment" animation_step="2">第二步显示</div>
```

### 5.3 交互动画

- **卡片悬停**: `translateY(-3px)` 或 `translateY(-5px)` 上浮效果
- **侧边条**: `width: 1.5px` → `width: 2px` 宽度变化
- **图标悬停**: 背景色变化 `group-hover:bg-blue-600 group-hover:text-white`

## 7. 使用说明

### 7.1 创建新页面步骤

1. 复制上述完整模板
2. 修改 `<title>` 标签内容
3. 填写 Header 区域的场景分类和主标题
4. 填写需求背景和业务痛点内容
5. 修改工作流步骤的图标和文字
6. 更新方案价值的数据和描述
7. 根据需要调整右栏的预览内容

### 7.2 动画步骤配置

通过 `animation_step` 属性控制元素出现顺序：
- `animation_step="1"`: 第一批显示（背景+痛点卡片）
- `animation_step="2"`: 第二批显示（解决方案+右侧预览）
- `animation_step="3"`: 第三批显示（方案价值）
